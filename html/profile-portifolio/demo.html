The container (parent div) gets display: flex. The items are the direct children. Every example below changes a CSS property on one of them.

Save this as grid.html, open it in your browser, and change the grid CSS to see how the layout reacts. 

<!DOCTYPE html>
<html>
<head>
  <style>
    .grid-container {
      display: grid;                             /* turns on grid */
      grid-template-columns: 1fr 1fr 1fr;        /* 3 equal columns */
      gap: 10px;
      padding: 15px;
      border: 2px dashed #ec4899;
      background: #fdf2f8;
    }

    .grid-item {
      background: #ec4899;
      color: white;
      padding: 20px;
      border-radius: 6px;
      text-align: center;
    }
  </style>
</head>
<body>

  <div class="grid-container">
    <div class="grid-item">1</div>
    <div class="grid-item">2</div>
    <div class="grid-item">3</div>
    <div class="grid-item">4</div>
    <div class="grid-item">5</div>
    <div class="grid-item">6</div>
  </div>

</body>
</html>
The grid container (parent) gets display: grid plus the column/row definitions. Each direct child automatically becomes a cell. Try changing grid-template-columns to 1fr 2fr, 100px 1fr 100px, or repeat(4, 1fr).

3. Spanning cells with grid-column / grid-row

Any item can span multiple columns or rows using grid-column: span N or explicit line numbers.

span 2 cols
1 col
span 2 rows
normal
span 2 cols
<div class="grid-container">
  <div class="grid-item wide">span 2 cols</div>
  <div class="grid-item">1 col</div>
  <div class="grid-item tall">span 2 rows</div>
  <div class="grid-item">normal</div>
  <div class="grid-item wide">span 2 cols</div>
</div>
.grid-container {
  display: grid;
  grid-template-columns: repeat(4, 1fr);
  grid-auto-rows: 80px;
  gap: 10px;
}

.wide { grid-column: span 2; }
.tall { grid-row: span 2; }

/* Explicit lines also work:
   grid-column: 1 / 3;  (from line 1 to line 3) */

4. Named grid-template-areas (full page layout)

Name regions of the grid with strings and assign elements to them. Perfect for page layouts.

Header
Sidebar
Main
Footer
<div class="page">
  <div class="header">Header</div>
  <div class="sidebar">Sidebar</div>
  <div class="main">Main</div>
  <div class="footer">Footer</div>
</div>
.page {
  display: grid;
  grid-template-areas:
    "header  header  header"
    "sidebar main    main"
    "footer  footer  footer";
  grid-template-columns: 1fr 2fr 1fr;
  grid-template-rows: 60px 200px 60px;
  gap: 10px;
}

.header  { grid-area: header; }
.sidebar { grid-area: sidebar; }
.main    { grid-area: main; }
.footer  { grid-area: footer; }

When to use Flexbox vs Grid

Flexbox — one-dimensional layouts

Use when you're arranging items in a single row OR a single column. Great for navbars, toolbars, centering content, and distributing space between items on one axis.

Logo
Home
About
Contact
<nav class="navbar">
  <div class="logo">Logo</div>
  <a href="#">Home</a>
  <a href="#">About</a>
  <a href="#">Contact</a>
</nav>
/* Classic navbar */
.navbar {
  display: flex;
  justify-content: space-between;
  align-items: center;
  padding: 15px 20px;
}

Grid — two-dimensional layouts

Use when you need rows AND columns to align. Grid excels at full-page layouts, image galleries, dashboards, and any design where both axes matter.

Quick Rules of Thumb

If you're thinking about one row or one column → reach for flex.
If you're thinking about the page as a whole, with rows AND columns aligning → reach for grid.
They compose beautifully: a grid cell can use flex inside it, and vice versa.

Side-by-side comparison

Flexbox

  • 1D layout (row or column)
  • Content-first
  • Items size based on content
  • Great for components
  • Best for: navbars, cards, buttons

Grid

  • 2D layout (rows + columns)
  • Layout-first
  • Define the structure up front
  • Great for page layouts
  • Best for: dashboards, galleries, full pages
